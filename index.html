Cimientos del Sistema y Captura de Datos en Tiempo Real

El vacío actual: Los estudiantes no cuentan con una herramienta digital que les permita reportar de manera rápida y confiable la presencia de jaurías o perros callejeros durante sus desplazamientos diarios.

Enfoque de Ingeniería Industrial: Optimización del flujo de captura de información para minimizar el tiempo de respuesta entre la detección del riesgo y el registro del incidente, garantizando datos precisos para la toma de decisiones.

BackLog

  • Como estudiante de la UPN, quiero autenticarme con mi correo institucional para validar mi identidad dentro de PetWatch.
  • Como usuario en tránsito, quiero registrar un reporte en menos de 10 segundos para alertar rápidamente una situación de riesgo.
  • Como sistema, quiero capturar automáticamente GPS, fecha y hora para garantizar trazabilidad geoespacial del evento.
  • Como usuario, quiero visualizar confirmación inmediata del reporte para asegurar que fue registrado correctamente.
  • Definition of Done (DoD):
    1. Autenticación con correo institucional funcional.
    2. Registro de reportes en menos de 15 segundos.
    3. Captura automática de GPS, fecha y hora.
    4. Almacenamiento en base de datos geoespacial operativo.
    5. Confirmación visual de envío exitoso.
    6. Disponibilidad del sistema mayor o igual a 95% en pruebas.

Sistema de Reputación y Validación Comunitaria

El vacío actual: Los reportes de los usuarios pueden contener errores, información desactualizada o incluso falsas alertas, reduciendo la confiabilidad de la plataforma.

Enfoque de Innovación e Ingeniería Industrial: Aplicar sistemas de control de calidad y mejora continua para garantizar la confiabilidad de los datos. La innovación se centra en la validación colaborativa de información mediante inteligencia colectiva.

BackLog

  • Como usuario, quiero validar reportes de otros para mejorar la calidad de la información del mapa.
  • Como usuario activo, quiero acumular reputación según la calidad de mis aportes.
  • Como sistema, quiero asignar un nivel de confianza a cada reporte según validaciones recibidas.
  • Como administrador, quiero filtrar reportes de baja confiabilidad para evitar ruido en el mapa.
  • Definition of Done (DoD):
    1. Sistema de validación comunitaria funcional.
    2. Cálculo automático de score de confianza.
    3. Sistema de reputación activo por usuario.
    4. Priorización de reportes según confianza.
    5. Depuración automática de reportes inactivos o inconsistentes mayor a 72h.

Ruteo Óptimo para Brigadas de Rescate y Zoonosis

El vacío actual: El proyecto plantea que el gobierno local optimice recursos, pero no define cómo interactúan las autoridades con el software para resolver el problema en campo.

Enfoque de Ingeniería Industrial: Optimización de la cadena de suministro de servicios sanitarios. Pasamos de alertar al ciudadano a solucionar el problema de raíz con el algoritmo Vehicle Routing Problem (VRP).

BackLog

  • Como operador de zoonosis, quiero visualizar zonas críticas para planificar intervención.
  • Como sistema, quiero generar rutas iniciales basadas en ubicación de reportes.
  • Como sistema, quiero optimizar rutas para reducir distancia y tiempo de atención.
  • Como gestor municipal, quiero asignar brigadas según carga operativa y cercanía.
    1. Generación de rutas base desde datos geográficos.
    2. Algoritmo de optimización de rutas operativo (heurístico o VRP).
    3. Reducción simulada de tiempo de desplazamiento ≥ 20%.
    4. Asignación de recursos por zona funcional.

Integración GovTech con Infraestructura Urbana

El vacío actual: PetWatch funciona de manera aislada en el ecosistema del smartphone.

Enfoque de Innovación: Conectar el software con el hardware de la ciudad de Cajamarca. Si la comunidad es una red de sensores, la ciudad debe reaccionar físicamente a ellos.

BackLog

  • Como sistema, quiero clasificar la severidad de los reportes para activar respuestas diferenciadas.
  • Como ciudadano, quiero que alertas críticas activen iluminación o señalización cercana.
  • Como sistema, quiero simular activación de cámaras o sensores urbanos mediante webhook.
  • Como usuario, quiero visualizar rutas seguras actualizadas en tiempo real.
    1. API webhook funcional para eventos críticos.
    2. Simulación de activación de luminarias/cámaras.
    3. Clasificación de severidad de incidentes.
    4. Actualización de rutas seguras en interfaz.
    5. Registro de respuesta del sistema urbano.
